01  Project kya hai

Ek web application jahan aap meeting ki audio/video upload karte ho aur AI automatically usse samajh ke useful cheezein nikaalta hai.

🎙️ Transcription

Audio → text, timestamps ke saath

🗣️ Speaker Diarization

"Kaun bola" — Speaker 1, 2, 3…

🧠 Intelligence

Summary, decisions, risks, action items

🔍 Semantic Search

Meaning se search, keyword se nahi

💬 RAG Chat

Meeting se sawaal poocho, citations ke saath

👥 Multi-tenant

Har organization ka data alag & secure

Core challenge jisne poori design decide ki: 1 ghante ki meeting process karne mein 5–10 minute lagte hain, lekin ek HTTP request sirf ~30 second zinda rehti hai. Isliye humne background workers (Celery) use kiye — API turant "le liya" bolta hai, worker aaram se process karta hai, frontend status poll karta hai.

03  Ek Meeting ka Safar (End-to-End)

Jab aap ek audio upload karte ho, ye 5 stages background mein chalti hain:

# User audio upload karta hai
   API turant meeting banata hai (status=uploaded) → 202 "le liya"
   Celery ko task bhejta hai (Redis queue mein)

# Worker background mein utha ke process karta hai:
   1. EXTRACTING   → FFmpeg: video/audio → 16kHz mono WAV
   2. TRANSCRIBING → Whisper: audio → text + timestamps
   3. DIARIZING    → Pyannote: "kaun bola" (optional)
   4. ANALYZING    → LLM: summary, decisions, action items
   5. EMBEDDING    → chunks → vectors → ChromaDB (search/chat ke liye)
   → COMPLETED

# Frontend har 3 second status poll karta hai → progress bar live
Clean Architecture (har request ka rasta): Endpoint (HTTP) → Service (business logic + transaction) → Repository (sirf SQL) → Database. Har layer sirf apna kaam jaanti hai — isse code testable, maintainable, aur swappable rehta hai.

04  12 Modules — Detail mein

Humne project ko 12 modules mein banaya — foundation pehle, features baad mein. Har module ke saath: kya banaya, kaunsa concept seekha, aur kya problem aayi.

M1

Scaffold + Docker + Config

Foundation
Kya banaya
FastAPI app factory, config (env variables), structured logging, error handling, health endpoints, Docker Compose (6 services: Postgres, Redis, MinIO, ChromaDB, backend, worker).
Concept
12-Factor config (settings env se, code se alag), fail-fast (production mein default password ho to app boot hi nahi hogi), liveness vs readiness health probes.
Problem
DB unreachable ho to health-check 20+ second hang karta tha. Fix: connect_timeout=5 add kiya — fail fast.
M2

Database Models + Migrations

Data layer
Kya banaya
16 SQLAlchemy tables (users, meetings, transcripts, speakers, insights, action items, chat…), mixins (UUID PK, timestamps, soft-delete), Alembic migrations.
Concept
Composition over inheritance (mixins), soft delete (data flag karo, delete mat karo), cascade strategy (speaker delete ho to transcript bache — SET NULL).
Problem (classic!):
Alembic ka auto-generated downgrade() tables to drop karta tha par PostgreSQL ENUM types nahi. Downgrade ke baad re-upgrade fail: "type user_role already exists". Fix: manually 8 enum drops add kiye. Isliye migrations hamesha review karo!
M3

Authentication (JWT + RBAC)

Security
Kya banaya
Signup, login, logout, forgot/reset password. JWT access token + refresh token rotation + reuse detection. Role-based access (Admin/Manager/Employee). Rate limiting.
Concept
3 token strategies: password→bcrypt (slow by design), access→JWT (stateless), refresh→SHA-256 hash (revocable). Refresh rotation: har use pe naya token, chori detect ho jaati hai. Anti-enumeration: "wrong password" aur "no user" ka same error.
Problem
Rate limiter (slowapi) ne har success response pe crash kiya — usse response parameter chahiye tha. Sirf jab rate limiting ON (production) tab dikha, tests mein nahi. Fix: har endpoint mein response: Response.
M4

File Upload + Storage

Provider Pattern
Kya banaya
Drag-drop upload, storage abstraction (LocalStorage ↔ S3/MinIO, same interface), file validation (magic bytes), streaming size cap, meeting CRUD, presigned download URLs.
Concept
Provider Pattern (ek interface, kai backends — env se switch), magic-byte validation (extension pe bharosa nahi — file ka actual header check), compensating transaction (DB fail ho to stored file delete).
Problem (star bug):
Size-limit wrapper sirf read() deta tha. LocalStorage test pass hua, par real MinIO pe boto3 ne .close() call kiya → crash. Seekh: real infra pe test karo, mock kaafi nahi.
M5

Audio Pipeline (Whisper)

AI starts
Kya banaya
FFmpeg audio extraction, transcription providers (local Whisper / OpenAI / stub), Celery pipeline task, status state machine, idempotent reprocess.
Concept
Strategy Pattern (3 transcribers, ek interface), thin task / fat service (Celery task patla, logic service mein — bina broker ke testable), state machine (illegal status jumps rukein), idempotency (retry pe duplicate na ho).
Smart trick
Stub provider banaya — poori pipeline bina 150MB model download ke test hoti hai. CI mein invaluable.
M6

Speaker Diarization

Graceful degradation
Kya banaya
Pyannote diarization (optional), overlap-matching (transcript segments ko speakers se link karna), speaker rename.
Concept
Graceful degradation (HF token nahi → skip, transcript phir bhi milega), overlap matching (Whisper aur Pyannote alag segment karte hain — max time-overlap se speaker assign), best-effort stage (diarization fail ho to bhi meeting complete).
Problem
Stub diarizer bhi ffprobe call karta tha (host pe ffmpeg nahi) — test mein wo patch karna bhool gaya. Fix: test fixture mein pin kiya.
M7

Meeting Intelligence (LLM)

Structured AI
Kya banaya
LLM se summary, executive summary, decisions, risks, action items, deadlines. Structured JSON output, OpenAI + stub providers.
Concept
Structured outputs (LLM ko Pydantic schema force karo — fragile text parsing nahi), core vs best-effort (analysis fail = pipeline fail, kyunki summary core feature hai), prompt engineering (low temperature, "only from transcript").
M8

RAG (Search + Chat)

ChromaDB
Kya banaya
Embedding providers, vector store (ChromaDB ↔ in-memory), chunking, semantic search, RAG chat with citations.
Concept
RAG = Retrieve relevant chunks → LLM ko context do → grounded answer. Dual write (vectors→Chroma, twin rows→Postgres — citations + cleanup). Tenant isolation (har query org-scoped). Citations (answer ke saath source + timestamp = verifiable AI).
Problem
Multi-key metadata filter ko ChromaDB $and syntax chahiye, in-memory store flat dict. Fix: translate helper.
M9

Frontend Foundation

Next.js
Kya banaya
Next.js 15 + TypeScript + Tailwind, UI components (ShadCN-style), typed API client with auto token-refresh, auth pages, app shell (sidebar+topbar), dashboard, dark mode.
Concept
Token refresh coalescing (concurrent 401s ek hi refresh share karein — backend reuse-detection se bachne ke liye), full-stack type safety (backend schema ↔ TS types), route groups.
M10

Upload UI + Meeting Detail

Live polling
Kya banaya
Drag-drop upload (XHR progress bar), meeting detail with tabs (Overview/Transcript/Speakers/Intelligence), live status polling, speaker rename, action-item toggle.
Concept
Live polling (pipeline chalte waqt har 3s refresh, terminal state pe stop), XHR upload progress (fetch nahi de sakta), optimistic UI (turant update, fail pe revert).
Problem
npm run build chalaya jab npm run dev chal raha tha → dono .next share karte hain → corrupt → 500. Fix: node kill, rm -rf .next, restart. Kabhi dono ek saath mat chalao.
M11

Chat UI + Search

RAG frontend
Kya banaya
Semantic search page (results + link + timestamp + match%), ChatGPT-style RAG chat (session sidebar + message thread + citations).
Concept
RAG chat UI (citations = trust layer), optimistic chat (user turn turant), auto-scroll to bottom. Aur beech mein Ollama provider add kiya — free local LLM.
M12

Dockerize + Deploy + CI/CD

Production
Kya banaya
Frontend multi-stage Dockerfile (313MB standalone image), production compose (all services, restart policies, resource limits, split workers), GitHub Actions CI, deployment docs.
Concept
Multi-stage build (image chhoti), NEXT_PUBLIC = build-time (browser var bundle mein bake), dev vs prod compose, non-root container, queue isolation (heavy transcription vs light exports alag workers).

08  Interview Q&A (30+)

Click karke answer kholo. Ye project ke har part se hain — architecture, AI, security, DevOps, frontend.

🏗️ Architecture & Design

Long-running AI task ko HTTP request mein kyun nahi chalate?
HTTP request ~30s mein timeout hoti hai, transcription minutes leti hai. Isliye task ko queue (Redis) mein daalte hain, Celery worker process karta hai, client status poll karta hai. API turant 202 lauta deta hai.
Clean Architecture layers kya hain aur faayda?
Endpoint (HTTP) → Service (business logic + transaction) → Repository (SQL) → DB. Har layer sirf apna kaam jaanti hai. Faayda: testable (service ko bina HTTP test karo), swappable (DB badlo, sirf repository change), maintainable.
Repository pattern kyun? Service ne commit kyun kiya, repository ne nahi?
Ek business operation (signup = org + user + audit) ek transaction hona chahiye — wo boundary service ke paas hai. Repository sirf queries deta hai, reusable rehta hai. Isse "ek operation = ek commit" enforce hota hai.
Provider / Strategy pattern kya hai, kahan use kiya?
Ek interface, kai implementations, env se switch. Storage (Local↔S3), transcription (Whisper↔OpenAI↔stub), LLM (Ollama↔OpenAI↔stub), vector store (Chroma↔memory). Faayda: vendor lock-in nahi, testing ke liye stub, config se change.
Idempotency kya hai, pipeline mein kyun zaroori?
Same operation dobara chale to same result, duplicate nahi. Celery task 3 baar retry ho sakta hai; humne segments/speakers delete-then-insert kiye taaki reprocess pe duplicate na ho.

🔐 Security & Auth

JWT stateless hai to logout kaise karte ho?
Access token revoke nahi kar sakte (isliye short-lived, 30 min). Refresh token server-side stored hai — usse revoke karte hain. Instant revocation chahiye to JWT denylist (Redis).
Refresh token rotation ka faayda?
Har refresh pe purana token burn, naya issue. Agar chori hua token dobara use ho (reuse detection), to hum us user ke saare sessions revoke kar dete hain. Chori automatically pakdi jaati hai.
Password bcrypt se, refresh token SHA-256 se — dono alag kyun?
Password low-entropy (guessable) hai — bcrypt slow-by-design brute-force rokta hai. Refresh token 384-bit random hai (brute-force impossible), aur exact-match lookup chahiye — SHA-256 fast + deterministic. bcrypt ka random salt exact lookup todh deta.
RBAC mein 401 vs 403 ka difference?
401 = "kaun ho?" (login missing/invalid). 403 = "pata hai kaun ho, par allowed nahi" (role kam hai).
Anti-enumeration kya hai?
Attacker ko ye pata na chale ki kaunsi email registered hai. Login mein "wrong password" aur "no user" ka same error; forgot-password hamesha "link sent" bolta hai.
File upload securely kaise validate karte ho?
Extension pe kabhi bharosa nahi (attacker-controlled). Magic bytes (file header) check karo — .mp3 naam wali .exe header pe pakdi jaati hai. Plus streaming size cap.

🤖 AI / ML Pipeline

RAG kya hai aur kyun use karte ho?
Retrieval-Augmented Generation. Poora corpus LLM ko nahi dete — question embed karke relevant chunks retrieve karte hain, phir wo context LLM ko dete hain. Faayda: sasta (kam tokens), current (naya data bina retrain), grounded (hallucination kam).
Vector DB aur relational DB dono kyun?
Vector DB (Chroma): similarity math. Relational (Postgres): truth + citations + cleanup. Twin rows orphan vectors rokte hain — meeting delete ho to Chroma bhi clean ho.
Whisper aur Pyannote alag segment karte hain — align kaise?
Time-overlap matching. Har transcript segment ke liye us speaker ko assign karo jiske turns us time-range se sabse zyada overlap karte hain. Ties deterministic tiebreak se.
LLM se reliable structured data kaise nikaalte ho?
Structured outputs / JSON mode — model ko Pydantic schema force karo. Text parsing (regex) fragile hai. Ollama mein humne JSON mode + schema-in-prompt + retry use kiya.
Hallucination kaise kam karte ho?
Low temperature (0.2), strict system prompt ("only from transcript"), structured constraints, aur RAG grounding (context ke bahar answer mat do). Citations se verifiability.
Optional AI feature (diarization) down ho jaye to?
Graceful degradation: null provider (ek code path) + best-effort stage (failure non-fatal). Core value (transcript + summary) kabhi block nahi hota.
Core feature (LLM analysis) fail ho to?
Pipeline fail hota hai (retry + FAILED with reason), kyunki summary core value hai. Par transcript pehle commit ho chuka — FAILED meeting bhi readable rehti hai.
Bahut lambi meeting (3 ghante) kaise handle karoge?
Chunk + map-reduce: har chunk summarize, phir combine. Ya bade context-window model. Abhi humne explicit truncation with marker use kiya (silent nahi).

⚙️ Celery / Infra

Worker transcription ke beech crash ho jaye to?
acks_late=True: task tab acknowledge hota hai jab poora ho. Worker mare to task queue mein wapas, dusra worker uthata hai. Idempotent design se safe.
Do queues (ai_pipeline, default) kyun?
1hr transcription ke peechhe 2s export stuck na ho. Heavy tasks alag queue, light alag — dono independently scale, transcription workers GPU pe.
Liveness vs readiness probe?
Liveness = "process up hai?" (fail → restart). Readiness = "traffic le sakta hai?" (dependencies check; fail → routing band, restart nahi).

🐳 Docker / DevOps

Docker image chhoti kaise karte ho?
Multi-stage build (build tools final image mein nahi), Next.js standalone output (sirf used modules), alpine base, non-root user. Hamara frontend 313MB.
NEXT_PUBLIC var build-time kyun bake hoti hai?
Browser mein chalti hai, isliye bundle mein inline hoti hai build ke waqt. Production mein build arg se real API URL pass karte hain.
Dev aur prod compose alag kyun?
Dev: hot reload, bind mounts, debugging. Prod: immutable images, restart policies, resource limits, infra ports hidden (security), multiple workers.
Container non-root kyun chalate ho?
Defense in depth — container compromise ho to root access nahi milta, blast radius kam.

💻 Frontend

JWT frontend mein kahan store karte ho, trade-off?
localStorage (reload survive) + in-memory mirror. Trade-off: XSS risk vs httpOnly cookie (CSRF risk). Short access TTL + refresh rotation se mitigate.
Long task ka status frontend pe kaise dikhate ho?
Polling (simple): interval pe status fetch, terminal state pe stop, cleanup on unmount. Scale pe WebSocket/SSE (server push) better.
Upload progress kaise dikhate ho?
XMLHttpRequest ka upload.onprogress (fetch support nahi karta). Real % milta hai.
Optimistic UI kya hai?
Server confirm se pehle UI update (snappy feel), fail pe revert. Chat message aur action-item toggle mein use kiya.

09  Concepts Glossary (quick revise)

ORM

Object-Relational Mapper — Python classes se DB use karo, raw SQL nahi (SQLAlchemy).

Migration

DB schema ka "git" — changes versioned files mein (Alembic).

JWT

JSON Web Token — signed token jisme user identity, stateless verify.

RBAC

Role-Based Access Control — role se permissions (Admin/Manager/Employee).

Embedding

Text → numbers ki list jo meaning capture karti hai; similar text paas hota vector space mein.

RAG

Retrieval-Augmented Generation — relevant context retrieve karke LLM ko do, grounded answer.

Diarization

"Kaun bola" — audio ko speakers ke hisaab se baantna.

Idempotent

Same operation dobara = same result, duplicate nahi.

Presigned URL

Time-limited signed link — browser directly storage se download kare, API bandwidth bache.

CORS

Browser rule — ek origin dusre ko request tab kare jab backend allow kare.

Soft delete

Row delete nahi, flag karo (recovery + audit).

Graceful degradation

Optional feature fail ho to core kaam chalta rahe.
